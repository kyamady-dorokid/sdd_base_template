## 壁打ち概要
Claude Code を Windows ホストで起動し、作業対象が WSL 上のファイル（`\\wsl.localhost\...` / `\\wsl$\...`）の場合、Windows 側からの操作が WSL の CLI/git と差分（改行 CRLF/LF・実行権限・所有権・symlink・inotify 監視・大文字小文字・性能）を生む懸念がある。これに対する注意喚起・回避誘導を SDD ベースルール層へ追加する。

## 決定事項
| # | 決定 | 理由 | 決定日 |
|---|---|---|---|
| 1 | 方式は **A: Markdown ベースルール追加のみ**。PreToolUse 等の機構強制(B)は実装しない | A で十分と合意済み | 2026-06-20 |
| 2 | トリガーは「**Windows 実行 かつ 作業対象が WSL パス**」の越境時のみ（Windows ネイティブでは出さない） | ノイズ回避・普遍性 | 2026-06-20 |
| 3 | 発火は**セッションで最初の自動操作の直前に1回だけ**。内容は差分懸念＋WSL内 `claude` 起動への切替誘導＋続行可否の確認 | 要件どおり | 2026-06-20 |
| 4 | 反映方式は **B（参照＋発火指示）**：`environment-boundary-policy.md` を新設し、SDD-BASE ブロックに「所在リンク」＋「発火指示1行」を追加 | 参照のみだと警告が実質発火しないため。要件の挙動を実機で満たす | 2026-06-20 |
| 5 | 配置は配布層 `payload/overlay/`：rules に md 新設、`snippets/CLAUDE.sdd.md`・`AGENTS.sdd.md` を**同一変更で同期** | 全リポへ配布される環境非依存層 | 2026-06-20 |
| 6 | 「セッション初回」判定はエージェントの自己追跡（文脈で判断）。要約で稀に再警告し得るが許容 | A 方式の限界として合意 | 2026-06-20 |

## 作業ルール（本変更に適用）

- TDD 対象外（ドキュメント変更）。**CLAUDE.md/AGENTS.md スニペットの同期**と**参照リンク切れなし**を確認。

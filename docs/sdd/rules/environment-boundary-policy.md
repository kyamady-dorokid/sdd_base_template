# 環境越境ポリシー（Windows × WSL）

Claude Code / Codex を **Windows 上で起動**し、作業対象が **WSL 上のファイル**
（`\\wsl.localhost\...` または `\\wsl$\...`）の場合に適用するベースルール。
環境非依存のベースルール層として全リポジトリへ配布される。

## なぜ注意が必要か
Windows 側からの直接ファイル操作は、WSL 上の CLI / git と次の点で**差分や不整合**を生む可能性がある。
- 改行コード（CRLF / LF）の混入
- 実行権限・所有権（パーミッション）の喪失・変化
- symlink（シンボリックリンク）の扱いの違い
- ファイル監視（inotify）が効かない／ホットリロード不発
- 大文字小文字の区別（case sensitivity）の差
- ファイルアクセスのパフォーマンス低下

## トリガー条件（越境時のみ発火）
次の **両方**を満たすときのみ発火する。
1. 実行環境が **Windows**（デスクトップ版、または VS Code の Windows ターミナル等）。
2. 作業対象ディレクトリが **WSL パス**（`\\wsl.localhost\...` / `\\wsl$\...`）。

> 「Windows だから常に」ではなく「**越境しているから**」発火させる。
> Windows ネイティブのプロジェクト（WSL を跨がない）や非 Windows 環境では**発火しない**（ノイズ回避）。

## タイミング
そのセッションで **最初の自動操作（ファイル変更・コマンド実行など）を実行する直前に、1回だけ**警告する。
以降の操作では繰り返さない。

## 警告に含める内容
1. 現在 Windows 上で起動しており作業対象が WSL ファイルである旨（差分懸念の説明）。
2. **回避策**: WSL 内のターミナルから `claude` を起動し、同じディレクトリで作業することを推奨。
3. **確認**: このまま Windows 側で続行してよいかをユーザーに尋ねる。

### 警告文言（ドラフト）

## 運用上の限界（本ポリシーは Markdown ルールのみ）
- 本ポリシーは**機構による強制（フック等）ではなく、エージェントが従う Markdown ルール**である。
- 「セッション初回かどうか」は**エージェントの自己追跡**（会話文脈で「警告済み」を判断）による。
  長時間セッションで文脈要約が走った場合など、稀に再警告することがあるが許容範囲とする。
- 環境判定（Windows か／WSL パスか）は確実な検出 API を前提にしない。判断材料（OS 種別、
  作業パスが `\\wsl.localhost` / `\\wsl$` で始まるか等）から越境の可能性が高いと判断したら発火する。
